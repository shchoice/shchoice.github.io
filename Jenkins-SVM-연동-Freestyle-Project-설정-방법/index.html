Jenkins - SVM 연동 > Freestyle Project 설정 방법

개요

  • Jenkins의 Freestyle Project는 소규모 프로젝트나 단일 브랜치를 관리할 때 간단한 빌드 및 배포 작업에 매우 유용
  • Jenkins와 SVM을 연동하여 Freestyle Project로 단일 브랜치 관리하여 자동으로 빌드를 유발하는 방법을 알아보자

설정 방법

1. Jenkins에서 Job 생성

  • Dashboard > New Item > Freestyle project 선택

  • 프로젝트 이름 작성 > OK 버튼 클릭

2. 구성(Configuration) > 일반 설정(General)

  • 옵션설명
    • Github 과 연동일 경우 Github Project 클릭
    • Throttle builds : 빌드를 동시에 실행하는 상황에서 빌드의 병렬 실행 수를 제한하거나 특정 노드에서만 실행하도록 제어하는 기능(일반적으로는 자주 사용되는 옵션은 아님)

3. 구성(Configuration) > 소스 코드 관리(Source Code Management)

  • 옵션 설명

    • Git을 선택하고, Repository URL에 프로젝트의 Git 주소를 입력

      • 참고사항

        시스템에 Git이 설치가 되어져 있어야 함, 안그러면 에러 발생

    • Branches to build에서 빌드할 브랜치를 선택합니다. 단일 브랜치만 관리할 경우, 특정 브랜치(ex: master)만 선택

      • 참고사항

        Add Branch를 통해서 */master 브랜치, */develop 브랜치, */jenkins-test 3개의 브랜치를 만들어 보았으나 마지막에 설정했던 */jenkins-test 브랜치만 동작, 따라서 1개씩 여러 프로젝트를 만들거나, multibranch pipeline 을 사용하자

    • Credentials : Git 서버 인증

      • Add Credentials

        • 세부 설정사항
          • Scope
            • Global : 이 자격 증명이 시스템 전체에서 사용될 수 있음
            • System : 특정 시스템 또는 프로젝트에서만 이 자격 증명을 사용할 수 있음
          • Username : 대상 서버 계정
          • Password : 대상 서버 계정의 패스워드
          • ID (선택사항) : 자격 증명에 대해 Jenkins에서 사용할 고유 식별자를 입력하는 옵션, 기본적으로 이 필드를 비워두면 Jenkins가 자동으로 ID를 생성

4. 구성(Configuration) > 빌드 유발(Build Triggers)

  • 옵션 설명

    • 빌드를 원격으로 유발 (예: 스크립트 사용)

      • 외부 시스템(스크립트, CI/CD 파이프라인 등)에서 Jenkins 빌드를 원격으로 트리거할 수 있음

      • HTTP 요청을 통해 Jenkins API를 호출하여 빌드를 유발할 수 있음

      • 외부 도구나 서비스에서 Jenkins 빌드를 트리거해야 하는 경우 사용됨

        ※ Yona SVM 같은 경우는 마이너한 제품이라 설정 필요

    • Build after other projects are built

      • 다른 프로젝트의 빌드가 완료된 후 현재 프로젝트의 빌드를 트리거하는 옵션
      • 업스트림/다운스트림 빌드 관계를 설정할 수 있음
      • 프로젝트 간 의존성이 있는 경우 사용

    • Build periodically

      • 주기적으로 빌드를 실행하는 옵션입니다. Cron 표현식을 사용

    • GitHub hook trigger for GITScm polling

      • GitHub Webhook을 통해 GitHub에서 커밋, PR(Pull Request) 등의 이벤트가 발생하면 Jenkins가 이를 감지하고 자동으로 빌드를 트리거하는 옵션

    • Poll SCM

      • SCM을 주기적으로 확인하여 변경 사항이 있을 때 빌드를 트리거하는 옵션, Cron 표현식을 사용하여 폴링 주기를 설정

      • Webhook이 지원되지 않는 환경에서 사용됨

※ 위 사진에서 설명이 안되어져 있는 부분은 별도의 플러그인 설치 후 생긴 것

5. 구성(Configuration) > 빌드 환경 (Build Environment)

  • 옵션 설명
    • Delete workspace before build starts
      • 빌드를 시작하기 전에 워크스페이스를 삭제하여 이전 빌드의 잔여물을 제거하고, 깨끗한 환경에서 빌드를 시작
      • 특히 이전 빌드의 결과물이 다음 빌드에 영향을 줄 수 있는 경우 유용
      • 프로젝트가 의존성 문제로 인해 실패할 가능성이 있는 경우, 이 옵션을 활성화하여 클린 상태에서 시작할 수 있음
      • 안정성과 일관성을 위해 자주 사용됨
    • Use secret text(s) or file(s)
      • 빌드 환경에서 비밀 정보(예: API 키, 비밀번호)나 파일을 사용하는 경우, 이를 안전하게 관리하고 사용할 수 있도록 설정
    • Add timestamps to the Console Output
      • 빌드 과정에서 시간 정보를 추적하여 성능 문제를 분석하거나 빌드 과정의 정확한 실행 시간을 확인
      • 빌드 시간을 분석하고 최적화할 때 많이 사용함
    • Inspect build log for published build scans
      • gradle 또는 Maven 프로젝트에서 빌드 스캔 기능을 사용할 때 유용
      • 빌드 성능, 문제 원인, 빌드 과정을 추적하고 분석할 때 유용
    • Terminate a build if it’s stuck
      • 빌드가 특정 시간 이상 실행되지 않고 멈춰 있는 경우 자동으로 빌드를 중단하는 옵션
    • With Ant
      • Apache Ant를 사용하는 프로젝트에서 빌드를 실행할 때 사용

6. 구성(Configuration) > 빌드 단계(Build Steps)

  • Spring 을 사용할 경우 gradlew 가 함께 SVM에 올라가져 있을 경우, 별도로 사용하지 않아도 gradle 빌드가 됨
  • 프로젝트에 맞는 빌드 환경에 맞게 설정

7. 구성(Configuration) > 빌드 후 조치(Post-build Actions)

  • 빌드가 성공적으로 완료된 후 알림이나 배포 작업을 설정할 수 있음

8. SVM에서의 설정 > Web hook

  1. Yona SVM 에서 설정 방법 : Yona > 프로젝트 이동 > 설정(톱니바퀴 모양) > 웹 후크

    • http://121.126.210.18:9091/multibranch-webhook-trigger/invoke?token=MY_SPECIAL_TOKEN

9. 빌드 결과 확인
