Jenkins 기본 개념 및 설치 & 기본 설정

설치 방법 (Docker 기반 환경이 아닌, 로컬 환경에 대한 수동 설치)

  1. 설치 환경 : CentOS 7

  2. Jenkins 패키지 설치 [참고]

  3. Jenkins 설정 및 기동

    • 설정 변경 : 포트

    • 기동

    • 초기 패스워드 확인

      • 방법 1 : sudo systemctl status -l jenkins
      • 방법 2 : vim /var/lib/jenkins/secrets/initialAdminPassword
      • 도커 : sudo docker exec ${CONTAINER_ID or CONTAINER_NAME} cat /var/jenkins_home/secrets/initialAdminPassword

    • 웹 페이지 접속 : localhost:9091

      • 초기 패스워드만 입력하면 됨

      • 플러그인 설치 : Install suggested Plugins 선택

  4. 설치 완료

계정 API Key 발급 받기

  • Jenkins에서 API Key 발급 받기

    • Dashboard > 내 계정 ID > 설정(Configure) > API Token

      • API Token은 앞으로 확인이 불가능하기 때문에 반드시 따로 관리해야함
        • API Token: 118d4bcca6443cc8549d52493539889e4b

  • 용도 : Jenkins API를 사용할 때, 인증을 위한 수단으로 사용됨, 비밀번호를 직접 사용하지 않고도 안전하게 인증을 처리할 수 있는 방법

    • 스크립트나 자동화 작업에서 인증

      • Jenkins에서 빌드 트리거나 Job 상태 조회, 로그 확인 등의 작업을 자동으로 처리해야 할 때 API Key를 사용해 인증
      • 이를 통해 비밀번호 대신 API Key를 통해 안전하게 인증 절차를 처리할 수 있음

    • REST API 호출

      • Jenkins는 REST API를 제공하며, 이를 통해 Jenkins의 여러 작업을 외부 애플리케이션에서 자동으로 제어할 수 있음
      • 예를 들어, Jenkins Job을 외부 시스템에서 트리거하거나 빌드 상태를 확인할 수 있음
      • API 호출 시, 사용자의 API Key를 함께 사용하여 인증

    • CI/CD 파이프라인에서 인증

      • Jenkins API Key는 외부 툴(예: GitHub Actions, Bitbucket Pipelines 등)과 Jenkins를 연동할 때 사용됨
      • 예를 들어, Jenkins에서 자동 빌드를 트리거하는 CI/CD 파이프라인에서 API Key를 사용하여 Jenkins에 접근할 수 있음

    • Webhook 인증

      • Webhook 요청을 보내거나 Jenkins에서 트리거된 이벤트에 대해 API Key를 사용해 인증된 요청을 보낼 수 있음
      • GitLab, GitHub 등의 소스 코드 관리 시스템에서 Jenkins API Key를 통해 Jenkins로 인증된 Webhook 요청을 보낼 수 있음

Debug 모드 사용

  • Jenkins를 설정할 때 문제점이 발생하면 디버그 모드를 통해 오류의 원인을 찾을 수 있기에 필요에 따라 설정을 하자 > Console Output 에서 확인 가능
  • Dashboard > Jenkins > System
    • Enable Debug Mode 활성화

Console Output

  • Dashboard > Project 명 > build 번호 > Console Output(콘솔 출력)

CSRF(Cross-Site Request Forgery) 설정 문제

  • yona 등 별도의 Jenkins 플러그인이 없는 외부 프로그래을 사용할 때 발생하는 문제

    • 다음과 같이 명령어를 보내면 CSRF 오류가 발생

    • 원인 : Jenkins는 CSRF 공격을 방지하기 위해 crumb (CSRF 보호 토큰)를 요청에 포함하도록 요구, 토큰이 포함되지 않은 요청은 Jenkins에서 거부

    • 해결 방법

      1. Crumb Token 을 포함하여 빌드 요청

        ※ Crumb Token 을 가져오는 방법

      2. Build Authorization Token Root 플러그인 설치

        • 설치 후 yona에서 웹 후크를 다음과 같이 변경하면 동작

        • Build Authorization Token Root Plugin이 CSRF 보호 우회를 가능하게 해주기 때문에 정상 동작

      3. Jenkins에서 CSRF Protection 기능 끄기 [참고] [비추]

        • sudo vim /usr/lib/systemd/system/jenkins.service

        • sudo systemctl daemon-reload

        • sudo systemctl daemon-reload

플러그인 설치

  • Dashboard > Jenkins > Plugins > Available plugins

    예시 ) Micorsoft Teams

  • 플러그인 설치 후 적용을 하려면 Jenkins Process를 재기동해야함
