MLflow 개요 및 설치법

배경

새로 들어오는 DL 프로젝트에 대해 매번 실험을 수기(PPT, 스프레드시트)로 수행하는 데에 한계를 느낌

  • 복수의 모델 및 여러 하이퍼 파라미터 의 값을 조정해 가며 테스트를 수행해 최적화된 모델을 산출하고 싶었음

    • Epoch, Layer Depth, Regularization(Dropout, Batch Norm, max_len_size, num_update_token 등)

  • 정확도 평가 지표를 여러개 사용하고 그 결과를 한눈에 보기를 희망

    • 분류 : Accuracy, Recall, Precision, F1-Score, AUROC 등
    • 예측 : MAE, RMSE, MPE, R2 Score 등

  • 산출한 모델의 버전 관리가 어려움

    • ex) model_ .pkl model_ .pb등

설치 환경

  • OS : Ubuntu 16.04
  • Python : 3.8
    • (python 3.6 이상이면 문제 없음)

MLflow 개요

  • MLflow란

    • MLflow는 실험(experimentation), 재현성(reproducibility), 배포(deployment) 및 Model registry 등의 ML life-cycle을 관리할 수 있는 오픈소스 플랫폼
    • CLI 및 웹 기반 GUI 를 지원
    • Tensorflow, PyTorch, Keras, sklearn, fast.ai 등 다양한 라이브러리와 통합하여 사용할 수 있음
    • 부분 유료인 WanDB 등 다른 툴도 존재하지만, 무료로 사용할 수 있음

  • 주요기능(크게 4가지 Component로 구성)

    1. MLflow Tracking
      • ML Code를 수행할 때 설정한 parameter, code version, metris, loss 의 변화, 시각화 자료 등의 log를 API 및 UI로 제공
      • Tensorflow & Keras, pytorch, FastAI, Scikit-learn, Spark, XGBoost 등의 라이브러리에 대해서는 자동으로 로그를 기록함(Automatic Logging)
    2. MLflow Projects
      • ML code를 재사용, 재구현 가능한 형태로 패키징하여 다른 데이터 과학자들과 공유하거나 프로덕션으로 변환 기능 제공
    3. MLflow Models
      • REST API 기반으로 실시간 서빙 및 추론 플랫폼 제공
    4. Model Registry
      • Centeralized model store의 역할을 수행함
      • Model의 생성부터 모델의 버전 및 스테이지를 충돌없이 관리할 수 있음
        따라서 Model Registry에 update가 발생할 경우, 핵심 정보를 로그로 남길 수 있도록 이벤트를 설계

  • REST API 및 CLI 를 통해 모든 기능에 엑세스할 수 있음

  • API는 Pyhthon, R, Java 등 존재

MLflow 설치 및 기동

  • MLflow 설치 및 확인

  • MLflow 디렉토리 생성

    • 간단한 구성을 위해 계정 home path 에 mlflow 디렉토리를 구성하고 해당 디렉토리에 mlflow의 artifact 및 data 파일을 저장

  • MLflow tracking server 기동

    • mlflow ui 로 기동하는 방법, mlflow server 로 기동하는 방법 총 2가지 방법이 있음

      • mlflow ui는 localhost에서만 기동 가능
      • mlflow server는 production 용도로 사용
      • worker를 여러개 띄울 수 있고 prometheus가 metrics를 가져갈 수 있는 엔드포인트를 제공하는 추가적 기능이 존재

    • mlflow ui로 기동 시 (작업 연속성을 위해서는 기동은 반드시 같은 디렉토리에서 수행해야함)

      • 옵션 설명

        • -h : 접속이 가능한 네트워크 주소
        • -p : 접속에 사용할 포트번호
        • –backend-store-uri
          • MLflow 엔티티가 기록되는 저장소
          • DB나 파일시스템으로 저장이 되며 아무설정 없을시 ./mlruns에 데이터가 로깅이 된다.
            • DB : sqlite:///path/to/file.db
            • 파일시스템 : file:///absolute/path/to/directory
        • –default-artifact-root
          • ML experiment에 대한 artifacts가 저장이 됨
          • DB나 파일시스템으로 저장이 되며 아무설정 없을시 ./mlruns에 데이터가 로깅이 된다.

    • mlflow server로 기동 시

      • 추가 옵션을 사용하고 싶을시 mlflow server –help 로 상세 확인

  • MLflow tracking server UI 확인

    • http://: 로 접속

    • tracking server를 띄었다면 ~/mlflow 에 mlruns라는 디렉토리가 생성된 것을 확인할 수 있음(Experiment artifacts 및 data)
