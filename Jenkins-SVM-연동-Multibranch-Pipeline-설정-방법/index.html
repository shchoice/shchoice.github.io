Jenkins - SVM 연동 > Multibranch Pipeline 설정 방법

개요

  • Jenkins Multibranch Pipeline을 사용하여 여러 브랜치의 빌드 및 테스트를 자동으로 관리하는 방법을 설명
  • Multibranch Pipeline은 각 브랜치를 자동으로 탐색하고, 브랜치나 PR이 변경될 때마다 이를 감지하여 빌드를 실행하는 강력한 기능을 제공
  • Jenkins를 활용한 Multibranch Pipeline 설정 방법을 단계별로 설명하며, 여러 브랜치 관리와 빌드 및 테스트를 자동으로 관리하는 방법을 소개

설정 방법

0. 플러그인 설치

  1. Multibranch Scan Webhook Trigger

    • Dashboard > Jenkins Management > Plugins > Available plugins 탭

1. Jenkins에서 Job 생성

  • Dashboard > New Item > Multibranch project 선택

  • 프로젝트 이름 작성 > OK 버튼 클릭

2. 구성(Configuration) > 일반 설정(General)

  • Optional 로 원하는 형태로 기술

3. 구성(Configuration) > Branch Sources

  • Add source > Git

    • Project Repository : Git 주소 작성

    • Behaviors

      • Discover branches : 리파지토리의 브랜츠들을 자동으로 탐색
      • Filter by name (with wildcards)
        • include : 브랜치들 중에서 원하는 브랜치들만 필터링하여 탐색함
        • exclude : 브랜치들 중에서 특정 브랜치를 제외하고 탐색함

    • Credentials : Git 서버 인증

      • Add Credentials

        • 세부 설정사항
          • Scope
            • Global : 이 자격 증명이 시스템 전체에서 사용될 수 있음
            • System : 특정 시스템 또는 프로젝트에서만 이 자격 증명을 사용할 수 있음
          • Username : 대상 서버 계정
          • Password : 대상 서버 계정의 패스워드
          • ID (선택사항) : 자격 증명에 대해 Jenkins에서 사용할 고유 식별자를 입력하는 옵션, 기본적으로 이 필드를 비워두면 Jenkins가 자동으로 ID를 생성

    • 참고사항 : 시스템에 Git이 설치가 되어져 있어야 함, 그렇지 않으면 에러 발생

4. 구성(Configuration) > Build Configuration

  • Multibranch pipelin은 Jenkinsfile로 브랜치를 관리 > 6번 항목에서 상세 기술

5. 구성(Configuration) > Scan Multibranch Pipeline Triggers

  • 브랜치 및 PR 변화를 감지하여 빌드를 트리거하는 방법을 설정하는 옵션

  • Scan by webhook

    • 외부 서비스(GitHub, GitLab 등)에서 변경 사항을 푸시하거나 PR이 발생할 때 자동으로 Jenkins에서 해당 변경을 스캔하고 빌드를 트리거할 수 있도록 설정하는 옵션
    • GitHub/GitLab Webhook을 통해 Jenkins가 즉각적인 변화를 감지
    • Yona 에서는 MR(PR)에 대해서는 즉각적인 변화를 감지하지 못함
    • Trigger token : Webhook에서 Jenkins로 요청을 보낼 때 인증을 위해 사용되는 토큰, 이 토큰을 사용하여 외부에서 Jenkins로 브랜치 스캔을 요청할 수 있으며 보안 목적으로 잘 관리되어야 함

  • Periodically if not otherwise run

    • 이 옵션을 활성화하면 Jenkins가 주기적으로 브랜치를 스캔하여 변경 사항이 있는지 확인
    • 변경 사항이 발견되면 빌드를 트리거
    • Yona의 경우는 즉각적인 MR(PR)이 되지 않기 때문에 해당 옵션을 활성화하여 주기적으로 변경 감지를 수행

6. Jenkinsfile 작성

해당 내용은 다음 페이지에서 자세하게 언급

7. SVM에서의 설정 > Web hook

  1. Yona SVM 에서 설정 방법 : Yona > 프로젝트 이동 > 설정(톱니바퀴 모양) > 웹 후크
    • http://121.126.210.18:9091/multibranch-webhook-trigger/invoke?token=MARINER5-RAG-API-Pipeline

8. 빌드 결과 확인

결론

  • Jenkins Multibranch Pipeline은 대규모 프로젝트에서 여러 브랜치를 효과적으로 관리하는 데 매우 유용
  • 각 브랜치마다 별도의 파이프라인을 설정하지 않아도 Jenkins가 자동으로 브랜치를 탐색하고 빌드를 실행할 수 있음
  • 특히 SVM 연동을 통해 자동화된 Webhook 설정으로 브랜치 변경 사항을 즉시 감지하고, 빠르고 효율적으로 빌드와 배포를 진행할 수 있음
  • Jenkins의 Multibranch Pipeline을 활용하면 복잡한 브랜치 전략을 더 쉽게 관리할 수 있으며, CI/CD 파이프라인을 통해 개발 속도를 높이고, 협업 효율성 또한 크게 향상시킬 수 있음
