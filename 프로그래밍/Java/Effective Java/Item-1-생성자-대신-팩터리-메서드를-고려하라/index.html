Item 1. 생성자 대신 팩터리 메서드를 고려하라

아이템 1. 생성자 대신 정적 팩터리 메서드를 고려하라

  • 핵심 정리

    • 장점

      • 이름을 가질 수 있음

        • 동일한 시그니처의 생성자는 2개 이상 가질 수 없음

      • 호출될 때마다 인스턴스를 새로 생성하지 않아도 된다(Boolean.valueOf)

        • 생성자를 내부/외부에서 여러개 생성할 수 있음

      • 반환 타입의 하위 타입 객체를 반환할 수 있는 능력이 있음(인스페이스 기반 프레임워크, 인터페이스에 정적 메소드)

      • 입력 매개변수에 따라 매번 다른 클래스의 객체를 반환할 수 있음(EnumSet)

      • 정적 팩터리 메서드를 작성하는 시점에는 반환할 객체의 클래스가 존재하지 않아도 됨 (서비스 제공자 프레임워크)

    • 단점

      • 상속을 하려면 public이나 protected 생성하기 필요하니 정적 팩터리 메서드만 제공하면 하위 클래스를 만들 수 없음
      • 정적 팩터리 메서드는 프로그래머가 찾기 어렵다.
