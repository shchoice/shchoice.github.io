VS Code에서 Git 사하기

  1. vscode에서 git에서 폴더 추가하면 git init 효과를 얻는다.

  2. commit이란 스테이징 영역의 변경사항을 레포지토리에 저장하는 행위, 변경사항을 묶어주는 단위

  3. 지금까지는 staging area에 저장을 하였고 깃 저장소에 저장을 해보겠다.

  4. 팔렛트- -> git add remote -> origin -> 원격 URL(https://github.com/shchoice/TIL.git) -> 푸시

  5. 로컬의 것을 업데이트 받고 싶으면 git pull이라고 팔렛트에 입력

  6. git bash

    • git status
    • git add 파일이름
    • git commit -m “urlretrieve example”
    • git push origin master

git remote 저장소 이름 변경

  • git 저장소 Settings 탭메뉴에서 Repository name을 변경합니다.
  • git bash를 통해 해당 프로젝트 로컬 경로로 이동합니다.이때, git remote -v라는 명령어를 이용하면, 현재 프로젝트 저장소 및 url를 출력해줍니다.
  • git remote set-url origin [변경된 url주소]명령어를 사용하여 저장소 url을 변경할 수 있습니다.
    ex) git remote set-url origin https://github.com/shchoice/TIL-Python-Web-Scraping
  • 최초 설정 시에는
    git remote add origin http://shchoice@133.186.171.23:9000/shchoice/OJT_DQ_Solution_Edu

Git 계정 변경

배경

  1. 컴퓨터 1대를 통해서 개인 Git을 위한 계정 그리고 별도의 팀 프로젝트를 위한 Git 계정을 동시에 사용할 일이 생김
  2. 공용 컴퓨터에 이미 설치된 Git 환경에서 작업을 수행할 경우가 생김

작업절차

  1. 현재 git에 등록된 계정 정보를 확인

  2. 변경할 계정을 등록 및 재확인

    • 이름에서 띄어쓰기 작업 시(ex. Seohwan Choi) “Seohwan Choi”으로 작성을 해야한다.

      만약에 “”로 감싸지 않는다면 이름에 2개 이상의 값이 있어 문제가 발생하게 된다. 추후 계정 변경 시 cannot overwrite multiple values with a single value 에러가 발생하게 된다. 해당 내용 발견 시 –replace-all 을 덧붙여 정정해야 한다.

  3. 윈도우 자격 증명 삭제

    • 제어판 > 사용자 계정 > 자격 증명 관리 > windows 자격 증명 > 일반 자격 증명
      • git:https://github.com/ 관련한 모든 자격 증명 제거
      • GitHub 관련한 모든 자격 증명 제거
